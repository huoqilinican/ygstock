1 为什么会产生库存超卖？

// 典型的错误代码（会产生超卖）
public function bugOrder() {
    // 步骤1：查询库存
    $stock = Db::name('goods')->where('id', 1)->value('stock');
    
    // 步骤2：判断库存
    if ($stock >= 1) {
        // 步骤3：更新库存
        Db::name('goods')->where('id', 1)->update(['stock' => $stock - 1]);
    }
}



并发执行时序图（假设库存初始为1）：

时间线    请求A                    请求B
  |      
  t1   查询stock=1               
  |      
  t2                            查询stock=1
  |      
  t3   判断：库存充足            
  |      
  t4                            判断：库存充足
  |      
  t5   更新：stock=0            
  |      
  t6                            更新：stock=0  ← 超卖！
  ↓



  问题根源：三个操作（读→判→写）不是原子的，在并发环境下会被拆散执行，导致多个请求基于相同的"旧数据"做出决策。




  2 数据库的ACID特性如何帮助我们？

  特性	                                作用	                      库存场景的应用
原子性(Atomicity)	          事务中的操作要么全成功，要么全失败       扣库存+创建订单必须同时成功或同时失败
隔离性(Isolation)	                 并发事务互不干扰	                       通过锁机制让修改串行化
持久性(Durability)	                提交后永久生效	                                  防止丢单
